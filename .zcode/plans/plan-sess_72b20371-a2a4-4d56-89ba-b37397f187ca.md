## طريقة إضافة مكتبة `quran_audio` إلى موقع التوثيق

### فهم آلية العمل
الموقع لا يخزّن محتوى المكتبات محلياً، بل يولّدها تلقائياً من ملف `README.md` و`pubspec.yaml` في مستودع GitHub الخاص بكل مكتبة. لذا إضافة مكتبة = **تعديل ملفّين فقط** (لا حاجة لكتابة أي محتوى يدوياً):

| الملف | الغرض |
|---|---|
| `packages.config.json` | سجلّ المكتبات (المصدر الوحيد للحقيقة) |
| `.github/workflows/deploy.yml` | خطّاف النشر (يتكرّر فيه اسم المكتبة في مكانّين) |

---

### التعديل 1: `packages.config.json`
إضافة هذا الكائن في نهاية المصفوفة (بعد `arabic_justified_text`):

```json
,
{
  "id": "quran_audio",
  "name": "quran_audio",
  "repo": "alheekmahlib/quran_audio",
  "branch": "main",
  "org": "alheekmahlib",
  "accent": "#113A38"
}
```

ملاحظات مهمة:
- **`id` = `quran_audio`** بالضبط — هو يُستخدم لبناء: اسم ملف البيانات (`src/data/quran_audio.json`)، ورابط pub.dev (`https://pub.dev/packages/quran_audio`)، واسم مشروع Cloudflare (`docs-quran-audio`).
- **`branch: "main"`** — يجب أن يطابق الفرع الافتراضي للمستودع وإلا سيفشل الجلب بـ 404.

### التعديل 2: `.github/workflows/deploy.yml` (مكانّان)

**(أ)** في قائمة الخيارات المنسدلة `options` (السطر 18–26) — إضافة `quran_audio` بعد `arabic_justified_text`:
```yaml
        options:
          - all
          - tafsir_library
          - hijri_date
          - flexible_sheet
          - floating_menu_expendable
          - floatica
          - moon_phase_plus
          - arabic_justified_text
          - quran_audio   # ← الجديد
```

**(ب)** في سلسلة fallback للنشر التلقائي عند push (السطر 54):
```yaml
            echo "list=tafsir_library hijri_date flexible_sheet floating_menu_expendable floatica moon_phase_plus arabic_justified_text quran_audio" >> $GITHUB_OUTPUT
```

---

### التحقّق محلياً (اختياري لكن موصى به)
بعد التعديلين، للتأكد أن المكتبة تُجلب وتُعرض بشكل صحيح:
```bash
node scripts/fetch-data.mjs quran_audio
PACKAGE_ID=quran_audio npx astro dev
```
هذا يولّد `src/data/quran_audio.json` (متجاهَل في git) ويشغّل خادم معاينة على `localhost:4321`.

### النشر
- يدوياً: من GitHub Actions → "Deploy Package Docs" → Run workflow → اختر `quran_audio`.
- تلقائياً: أي push إلى `main` يلمس `scripts/**` أو `src/**` سينشر كل المكتبات بما فيها الجديدة.
- مشروع Cloudflare Pages باسم `docs-quran-audio` يُنشأ تلقائياً عند أول نشر.

---

### ما لا تحتاج فعله
- ❌ لا كتابة ملفات markdown — المحتوى يُجلب من README.
- ❌ لا تكوين شريط جانبي — يُولّد من أقسام README (##).
- ❌ لا تعديل أي تخطيط/كومبوننت — كل المكتبات تشترك في نفس القالب.
- ⚠️ لا تُرتِب `src/data/quran_audio.json` — مُولّد تلقائياً ومتجاهَل في git.

### ما سأنفّذه
سأقوم بالتعديلين 1 و 2 أعلاه فقط (ملفّان، ثلاثة مواضع). لا ألمس أي شيء آخر.
